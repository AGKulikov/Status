# Единый мод Status Widget + Яндекс Навигатор

Эта сборка создаёт **один APK** с Android package `ru.natro.statuswidget`.
Он подписывается стабильным ключом Status Widget и поэтому устанавливается как обновление
HA1058 и последующих сборок.

Исходный APK Навигатора не хранится в Git. Скрипт принимает локальный файл пользователя и
проверяет его SHA-256:

`a529f4f180ce42e29b1b7b7d801d21428ed8801d65271c46b9a29cb5769b4c3b`

Поддерживаемая база: `YN_MonjaroMOD 29.4.2 M+mHUD+mDrive+mNavi+minimod 1.01`.

## Устройство сборки

- оригинальные ресурсы и нативные библиотеки Навигатора остаются главным пакетом ресурсов
  `0x7f`;
- Status Widget собирается с отдельным package id ресурсов `0x80`;
- его APK добавляется как внутренний ресурсный контейнер, а DEX добавляются в общий APK;
- два manifest-ресурса Status (описание accessibility service и FileProvider paths) копируются
  в корневую таблицу Навигатора; это позволяет AAPT2 связать манифест без недопустимых прямых
  ссылок из `0x7f` на вложенный пакет `0x80`;
- `NaviApplication` получает два ранних вызова: подключение ресурсов Status в
  `attachBaseContext()` и запуск сервисов Status в `onCreate()`;
- каждый импортированный экран Status подключает тот же ресурсный контейнер к собственному
  `AssetManager` до выполнения `onCreate()`; это покрывает отдельный landscape ResourcesImpl
  на Android 9;
- manifest получает HOME, сервисы, ресиверы и доступы Status, но Android-идентичность остаётся
  `ru.natro.statuswidget`;
- package-зависимые authorities провайдеров переносятся на `ru.natro.statuswidget`, поэтому
  исходный APK Навигатора не блокирует установку объединённого мода конфликтом провайдеров;
- единственный некорректно декомпилируемый `android:gravity="0x0"` нормализуется перед обратной
  сборкой; без этого именно эта база не проходит AAPT2;
- плавающее окно из этого мода читает `overlay_prefs`. Так как HOME и Навигатор теперь имеют
  один UID, лаунчер напрямую задаёт координаты и размер живой панели без второго приложения.

## Команда

Сначала Status APK должен быть собран с ресурсами `0x80`:

```bash
./gradlew -PembeddedNavigatorResources assembleGeelyRelease
```

Затем:

```bash
bash tools/navigator-mod/build-merged-mod.sh \
  /path/to/YN_MonjaroMOD.apk \
  /path/to/status-widget-0x80.apk \
  /path/to/apktool_3.0.2.jar \
  /path/to/status-widget-navigator.apk
```

Для подписанной сборки задаются `APKSIGNER`, `ZIPALIGN`, `KEYSTORE_FILE`, `KEY_ALIAS` и
`KEY_PASSWORD`. Приватный ключ не кладётся в репозиторий.

## Важная проверка на ГУ

Это смена package id у закрытого Навигатора, а не обычная перекомпиляция. До публикации как
основной релиз на магнитоле проверяются: холодный старт, загрузка карты, GPS, построение и
ведение маршрута, голос, офлайн-карты, вход в Яндекс, панель HOME, изменение размера панели,
разворот и возврат крестиком. При несовместимости конкретной проверки package name её нужно
точечно исправить в профиле этой версии APK, а не отключать проверки во всём приложении.
