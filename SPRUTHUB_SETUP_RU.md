# Настройка прямого Sprut.hub-коннектора

Status Widget HA подключается к Sprut.hub напрямую по WebSocket: MQTT-брокер и Home
Assistant для этой связи не нужны. Приложение получает каталог комнат, устройств, сервисов и
характеристик, позволяет выбрать конкретное значение и может отправлять команду в доступную для
записи характеристику.

## 1. Что потребуется

- сетевой маршрут от магнитолы до Sprut.hub;
- точный WebSocket URL, обычно вида `ws://192.168.1.2/spruthub`;
- email и пароль учётной записи Sprut.hub;
- серийный номер хаба, если в учётной записи их несколько.

Для `ws://` используйте только доверенную локальную сеть или VPN. Для доступа через внешнюю
сеть нужен `wss://` с сертификатом, которому доверяет Android магнитолы. Не публикуйте
незащищённый WebSocket-порт хаба в интернет.

Постоянный токен вручную вводить не нужно: коннектор получает временный сессионный токен при
авторизации и держит его только в активном соединении. Пароль хранится через Android Keystore
и не включается в экспорт настроек. После холодной загрузки доступ к секрету может появиться
только после первого разблокирования Android; служба перечитает настройки после разблокировки.

## 2. Подключение и проверка

1. В приложении откройте **Автоматизация и Home Assistant → Sprut.hub: подключение и выбор
   устройств**.
2. Включите коннектор и укажите полный WebSocket URL, email и пароль.
3. Если хаб один, серийный номер можно оставить пустым: приложение попытается определить его из
   списка. Если хабов несколько, укажите нужный серийный номер явно.
4. На магнитоле обычно стоит включить **Поддерживать соединение при погасшем экране**.
5. Нажмите **Сохранить и подключиться** и дождитесь строки
   **ONLINE — актуальный снимок получен**.
6. Нажмите **Обновить устройства**. Успешная проверка — список комнат, устройств и текущих
   характеристик обновился без ошибки.

Это и есть фактический тест соединения: одного сохранения настроек недостаточно. Старый каталог
может быть показан из локального кеша, поэтому подтверждением актуальной связи служат именно
`ONLINE` и успешное ручное обновление.

## 3. Добавление значения без ручного ID

После синхронизации найдите устройство по комнате, названию или характеристике. У каждой
читаемой характеристики показаны текущее значение, единица измерения и путь `aId/sId/cId`.
Вводить этот путь вручную не требуется:

- **В основную строку** создаёт информационный кирпичик с выбранной характеристикой;
- **Во всплывающий** создаёт отдельную плитку второго оверлея;
- **Готовая плитка** у сервиса подбирает основную характеристику, локальную иконку, подписи,
  цветовые правила и, когда это однозначно и безопасно, действие.

Пресеты распознают типовые сервисы ворот/штор, света, розетки, выключателя, замка, климата,
температуры, влажности, движения, протечки, дыма, вентилятора, клапана, охраны, батареи и
другие. Если метаданные нестандартного устройства неполны, будет создан нейтральный вариант —
его оформление и привязки можно изменить в редакторе элемента.

Цвет не читается «из кода Sprut.hub». Сырое значение поступает от хаба, а текст, цвет,
иконка, фон и видимость вычисляются локальными правилами Status Widget HA. Поэтому правила
можно менять независимо от устройства.

## 4. Управление устройством

Команда разрешена только когда:

- коннектор получил актуальный снимок;
- выбранная характеристика существует и помечена как доступная для записи;
- плитка не находится в `stale`, не заблокирована сценарием и не ожидает предыдущую команду.

Для логической характеристики готовая плитка обычно получает действие `TOGGLE`. Для `SET`
нужно указать конкретное допустимое значение. Если безопасное значение нельзя вывести из
каталога однозначно, готовый пресет остаётся только информационным до ручной настройки действия.
При необходимости включите локальное подтверждение нажатия в редакторе плитки.

Ответ API означает, что хаб принял команду, а не что физическое устройство уже завершило
движение. Окончательный статус приходит отдельным событием; если события нет, приложение
запрашивает резервный снимок.

## 5. Актуальность после перезагрузки и офлайна

При обрыве связи значения Sprut.hub помечаются как неактуальные, а действия блокируются. Старое
значение может отображаться со своим `staleText`/`staleColor`, но не считается свежим.
Коннектор автоматически переподключается, заново авторизуется и получает полный снимок. События,
пришедшие во время загрузки снимка, применяются перед переходом в `ONLINE`.

Таким образом, актуальность не зависит от того, менялось ли устройство в момент, когда
магнитола была выключена. Кеш каталога нужен для экрана выбора устройств, но сам по себе не
снимает состояние `stale`.

## 6. Сценарий HA → плитка Sprut.hub

Источник условия и целевая плитка не обязаны принадлежать одному коннектору. Например,
`input_boolean.show_gate_in_car` из прямого HA-коннектора может показывать плитку ворот, чей
статус и действие идут напрямую через Sprut.hub.

Сначала создайте всплывающую Sprut.hub-плитку и запомните её **Automation ID**, например
`sprut_gate`. Затем откройте **Сценарии: условия и видимость между коннекторами → Добавить** и
задайте:

- источник: `HOME_ASSISTANT`;
- ID подключения: `default`;
- ID ресурса: `input_boolean.show_gate_in_car`;
- путь к значению: пусто, чтобы использовать основное состояние сущности;
- оператор: `EQUALS`, operand: `on`;
- цель: `POPUP`, ID цели: `sprut_gate`;
- поле: `VISIBLE`, логическое значение: включено.

Когда HA передаст `on`, локальный движок покажет Sprut.hub-плитку. При `off`, потере HA или
неактуальном источнике обычное сравнение не совпадёт и плитка скроется. Сценарий меняет только
локальный интерфейс: состояние ворот продолжает читать Sprut.hub, а нажатие плитки по-прежнему
отправляется в Sprut.hub.

Точно так же источник MQTT может управлять элементом HA, а источник Sprut.hub — штатным
кирпичиком. Для сложных правил доступны расширенный JSON, режимы `ALL`/`ANY` и несколько
условий/действий.

## 7. Ограничение совместимости

Реализация содержит основной и совместимый варианты авторизации и проверяется unit-тестами на
поддерживаемых JSON-ответах. Однако формат прямого API может различаться между версиями
Sprut.hub, прошивками и нестандартными плагинами. Поэтому окончательная совместимость
подтверждается только на вашем хабе шагами из раздела 2.

Если `ONLINE` не появляется:

- проверьте, открывается ли указанный IP/порт из той же сети, где находится магнитола;
- проверьте точный путь `/spruthub`, схему `ws://` или `wss://` и доверие к сертификату;
- перепроверьте email, пароль и серийный номер;
- не принимайте видимый кешированный список за успешное подключение;
- сохраните текст состояния подключения и версию/ревизию хаба для диагностики совместимости.
