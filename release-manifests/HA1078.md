## Included fixes

- Apply the complete configured status-row geometry before the first
  `WindowManager.addView`, avoiding the oversized Android 9 startup frame.
- Populate the media row on its first visible frame and keep empty duration
  and progress elements out of startup measurement.
- Include duration, progress, source spacing and Home Assistant bricks in the
  configured minimum-height calculation without imposing a clipping fixed
  window height.
- Keep Sprut.hub smart-home commands on their writable target characteristics
  while driving labels, tint and active backgrounds from authoritative current
  characteristics.
- Automatically reconcile previously saved garage tiles from
  `TargetDoorState` to `CurrentDoorState`, so the live sequence is
  `Закрыто → Открывается → Открыто → Закрывается → Закрыто`.
- Refuse to present a target-only cover value as confirmed physical state when
  the device exposes no readable `Current*` characteristic.
