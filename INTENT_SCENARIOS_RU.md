# Команды устройств по Android Intent

Status Widget может выполнить один заранее сохранённый сценарий по Android Broadcast. Это
подходит для кнопок руля, лаунчера магнитолы и Home Assistant Companion: внешнее приложение
передаёт только имя намерения, а устройство, характеристика и значение берутся из защищённой
настройки Status Widget.

## Создание сценария

1. Включите основной Status Widget и настройте прямой коннектор Sprut.hub до состояния `ONLINE`.
2. Откройте **Автоматизация → Локальные сценарии → Намерения руля**.
3. Нажмите **Добавить** и задайте уникальный ID, например `parking_gate`.
4. Введите понятный префикс action, например `sh.car.parkovka`. При сохранении приложение
   обязательно добавит к нему случайный секретный суффикс вида `.x<32 hex>`.
5. Выберите устройство Sprut.hub, сервис и характеристику с признаком записи.
6. Выберите доступное действие:
   - `Включить` / `Выключить` для boolean;
   - конкретный режим из `validValues`, например `Открыть` или `Закрыть`;
   - `Переключить`, только если у характеристики действительно два состояния;
   - `Установить значение` для температуры, яркости, позиции или текста.
7. Сохраните сценарий.

Число проверяется по типу, `minValue`, `maxValue`, `minStep` и `validValues` как при сохранении,
так и непосредственно перед записью. Для boolean в редакторе используются `0/1`, а по протоколу
Sprut.hub отправляется корректный `boolValue: false/true`. Команда не выполняется по старому
кешу: приложение сначала ждёт актуальный снимок выбранного коннектора.

## Настройка кнопки руля

Для обычного рабочего режима укажите:

- тип: Android Broadcast;
- целевое приложение/package: `ru.natro.statuswidget`;
- action: ровно полное имя из карточки созданного сценария, включая секретный суффикс,
  например `sh.car.parkovka.x7f3e10a96b83490892e6c7248811d120`;
- extras: не нужны.

Имя сравнивается с учётом регистра. Одно включённое имя action может принадлежать только одному
сценарию. Повторные одинаковые сигналы в течение 750 мс и повтор во время уже выполняющейся
команды отбрасываются.

Обычный Android Broadcast не содержит надёжно проверяемой личности отправителя на Android 9–12.
Поэтому полный action содержит созданный приложением 128-битный секрет и работает как ключ
доступа: не публикуйте его и копируйте в систему кнопок руля целиком. Любое установленное
приложение, узнавшее полный action, теоретически может отправить такой же пакетный Broadcast.
Устанавливайте на магнитолу только доверенные приложения. Target package ограничивает адресата,
но сам по себе не является аутентификацией отправителя.

По этой же причине управляющие Intent-правила не включаются в экспорт настроек и пользовательские
пресеты. Импорт компоновки сохраняет уже существующие локальные правила на магнитоле, но не
переносит ключи ворот или замков на другое устройство. На новой магнитоле создайте правила заново
и скопируйте новые полные action в настройки кнопок руля.

Произвольные action Android разрешает регистрировать динамически, поэтому этот простой вариант
работает, пока foreground-служба Status Widget запущена. На магнитоле это штатный режим: служба
автоматически поднимается после загрузки, если виджет включён.

## Резервный запуск после выгрузки процесса

Android не умеет объявлять произвольный action из настроек в манифесте приложения. Если источник
умеет передавать extras, используйте фиксированный вход:

- package: `ru.natro.statuswidget`;
- action: `ru.natro.statuswidget.SCENARIO_TRIGGER`;
- строковые extras: `trigger_id=parking_gate` и показанный в карточке правила
  `trigger_token=<32-значный токен>`.

Если источник умеет задавать explicit component/class, можно передать то же полное секретное
имя action и указать класс `dezz.status.widget.automation.ScenarioTriggerReceiver`. Оба резервных
входа сначала проверяют точное совпадение с включённым сохранённым сценарием и только затем
запускают неэкспортируемую службу. Для фиксированного входа обязательны оба поля:
`trigger_id` и отдельный `trigger_token` из карточки правила.

Пример для Home Assistant Companion в обычном режиме:

```yaml
action: notify.mobile_app_имя_магнитолы
data:
  message: command_broadcast_intent
  data:
    intent_package_name: ru.natro.statuswidget
    intent_action: sh.car.parkovka.x7f3e10a96b83490892e6c7248811d120
```

Пример фиксированного резервного входа:

```yaml
action: notify.mobile_app_имя_магнитолы
data:
  message: command_broadcast_intent
  data:
    intent_package_name: ru.natro.statuswidget
    intent_action: ru.natro.statuswidget.SCENARIO_TRIGGER
    intent_extras: >-
      trigger_id:parking_gate,trigger_token:0123456789abcdef0123456789abcdef
```

## Поведение при отсутствии связи

Намерение не содержит команды Sprut.hub и не может подменить сохранённое значение через extras.
Если коннектор ещё восстанавливает актуальный снимок, событие ждёт готовности не более 15 секунд.
После отправки сетевой сбой не приводит к автоматическому повтору: для физической команды, особенно
`Переключить`, повтор после потерянного подтверждения был бы небезопасен. Неизвестные, выключенные,
нецелевые и слишком частые намерения игнорируются.
